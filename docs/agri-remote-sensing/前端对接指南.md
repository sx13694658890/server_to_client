# TiTiler 服务 — 前端对接指南

本文说明浏览器 / 前端应用如何调用本仓库提供的 **COG（Cloud Optimized GeoTIFF）** 切片服务。服务端基于 [TiTiler](https://developmentseed.org/titiler/) 的 `TilerFactory`，与官方 [Getting Started](https://developmentseed.org/titiler/user_guide/getting_started/) 行为一致。

## 1. 基础信息

| 项 | 说明 |
|----|------|
| 默认基址（本地） | `http://127.0.0.1:8008` |
| 交互式 API 文档 | `{基址}/docs`（Swagger UI） |
| OpenAPI JSON | `{基址}/openapi.json` |
| CORS | 已允许任意来源（`allow_origins=["*"]`），便于本地前端调试；生产环境建议改为明确域名列表 |

下文用 `{BASE}` 表示服务基址（勿以 `/` 结尾），例如 `http://127.0.0.1:8008`。

## 2. 数据源参数 `url`（必读）

几乎所有读图接口都通过 **查询参数 `url`** 指定数据源，值为 **TiTiler 能打开的路径或 URL**：

- **本机文件（仅当浏览器能访问到同一台机器上的服务时）**：`file:///绝对路径/影像.tif`（三段斜杠，路径需 URL 编码）
- **推荐（前后端分离 / 多用户）**：`https://...` 或 `http://...` 指向可匿名或通过签名访问的 COG；对象存储预签名 URL 亦可

前端拼接时请使用 **`encodeURIComponent(url)`**，避免 `&`、`?` 等破坏查询串。

示例（逻辑示意）：

```text
{BASE}/preview?url=file:///Users/demo/data/image.tif
```

等价于前端：

```js
const rasterUrl = 'file:///Users/demo/data/image.tif';
const preview = `${BASE}/preview?url=${encodeURIComponent(rasterUrl)}`;
```

> **注意**：若前端页面跑在用户电脑上，而 TiTiler 跑在服务器上，服务器上的 `file:///...` 指向的是 **服务器磁盘**，不是用户本机路径。跨环境请使用 HTTP(S) 可访问的地址。

## 3. 常用 HTTP 接口

以下路径均相对于 `{BASE}`，且通常需附带 `?url=...`（及可选参数）。

### 3.1 元数据

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/info` | 数据集基本信息（波段、数据类型等） |
| GET | `/info.geojson` | 同上，GeoJSON Feature（含范围几何） |
| GET | `/statistics` | 统计信息（依赖数据与查询参数） |

### 3.2 瓦片与 TileJSON（地图叠加核心）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/tiles` | 列出当前数据可用的瓦片集（TileMatrixSet）等 OGC 风格描述 |
| GET | `/tiles/{tileMatrixSetId}` | 某一瓦片集的元数据 |
| GET | `/tiles/{tileMatrixSetId}/{z}/{x}/{y}` | 默认格式栅格瓦片 |
| GET | `/tiles/{tileMatrixSetId}/{z}/{x}/{y}.{format}` | 指定格式，如 `png`、`jpg`、`webp`、`tif` 等（以 `/docs` 为准） |

Web 地图最常用的瓦片集标识为 **`WebMercatorQuad`**（与多数底图一致）。

**单块瓦片 URL 示例：**

```text
{BASE}/tiles/WebMercatorQuad/15/5412/12463.png?url=...
```

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/{tileMatrixSetId}/tilejson.json` | 返回 [TileJSON](https://github.com/mapbox/tilejson-spec)：含 `bounds`、`minzoom`、`maxzoom`、`tiles` 模板 URL 等，**推荐前端先请求此接口再配图** |

常用查询参数（部分，完整列表见 Swagger）：

- `tilesize`：瓦片像素边长，常见 `256` 或 `512`（需与地图库 `tileSize` 一致）
- `bidx`：波段索引（多波段时筛选显示波段）
- 渲染相关：`rescale`、`color_formula`、`colormap_name` 等（见官方 [Rendering](https://developmentseed.org/titiler/user_guide/rendering/)）

**TileJSON 示例：**

```text
{BASE}/WebMercatorQuad/tilejson.json?url=...&tilesize=256
```

### 3.3 预览图

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/preview` | 整幅预览图（默认尺寸/格式由服务端决定） |
| GET | `/preview.{format}` | 指定格式预览 |
| GET | `/preview/{width}x{height}.{format}` | 指定宽高与格式 |

适合缩略图、详情页静态图，不适合大范围缩放浏览（请用瓦片）。

### 3.4 点查值

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/point/{lon},{lat}` | 给定经纬度（WGS84，可通过其他参数指定坐标系）读取像元值 |

示例：`{BASE}/point/116.4,39.9?url=...`

### 3.5 内置网页预览（可选）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/{tileMatrixSetId}/map.html` | TiTiler 自带的简单地图页，便于快速验收数据源 |

## 4. 前端集成示例

### 4.1 先取 TileJSON，再 fitBounds + 瓦片层（Leaflet）

与 [TiTiler 文档中的 Leaflet 示例](https://developmentseed.org/titiler/user_guide/getting_started/#creating-a-web-map-with-leaflet) 相同思路：`bounds` 为 `[minX, minY, maxX, maxY]`（经纬度顺序），Leaflet 需要 `[[south, west], [north, east]]`。

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />
  <script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
</head>
<body>
  <div id="map" style="width: 100%; height: 600px;"></div>
  <script>
    const BASE = 'http://127.0.0.1:8008';
    const rasterUrl = 'file:///绝对路径/你的影像.tif'; // 生产环境请改为 HTTPS 可访问的 COG 地址

    const map = L.map('map').setView([0, 0], 2);
    L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
      maxZoom: 19,
      attribution: '&copy; OpenStreetMap'
    }).addTo(map);

    const tileJsonUrl =
      BASE + '/WebMercatorQuad/tilejson.json?tilesize=256&url=' + encodeURIComponent(rasterUrl);

    fetch(tileJsonUrl)
      .then((r) => r.json())
      .then((data) => {
        if (!data.bounds) return;
        const b = data.bounds;
        map.fitBounds([[b[1], b[0]], [b[3], b[2]]]);
        L.tileLayer(data.tiles[0], {
          tileSize: 256,
          opacity: 0.85,
          maxZoom: data.maxzoom ?? 22
        }).addTo(map);
      })
      .catch(console.error);
  </script>
</body>
</html>
```

### 4.2 MapLibre GL JS（思路）

1. `fetch` 同上 TileJSON URL。  
2. 将返回的 `tiles[0]` 作为 `tiles` 数组中的一项传入 `map.addLayer` 的 `raster` 源（注意 TileJSON 里可能带完整 query，勿重复丢失 `url=`）。  
3. `maxzoom`、`bounds` 与 Leaflet 类似，用于设置 `maxzoom` 与 `fitBounds`。

具体字段名以 MapLibre 当前 API 为准，核心是 **瓦片 URL 模板必须与 `tilesize`、query 一致**。

### 4.3 仅用 `<img>` 显示预览

```html
<img
  alt="preview"
  src="http://127.0.0.1:8008/preview.png?url=file:///绝对路径/你的影像.tif"
/>
```

（路径仍需 `encodeURIComponent` 若写在 JS 里。）

## 5. 调试与排错

| 现象 | 建议 |
|------|------|
| 浏览器控制台 CORS 报错 | 确认服务端已启用 `CORSMiddleware`（本仓库已加）；修改后需重启进程 |
| 404 / 无法打开数据集 | 检查 `url` 是否在 **运行 TiTiler 的机器** 上有效；`file://` 路径必须存在且可读。若曾使用 Docker 官方镜像的 **`/cog/...` 前缀**，本服务为根路径 **`TilerFactory`**，TileJSON 应为 **`{BASE}/WebMercatorQuad/tilejson.json`**（**无** `/cog`）。 |
| 有请求但地图空白 | 确认影像为 **Cloud Optimized GeoTIFF**；可尝试调整缩放级别；查看单瓦片 URL 是否 200 |
| 瓦片错位/模糊 | `tileMatrixSetId` 是否与客户端一致；`tilesize` 与地图库配置是否一致 |

更多说明见 TiTiler 文档：[Troubleshooting](https://developmentseed.org/titiler/user_guide/getting_started/#troubleshooting-common-issues)。

## 6. 安全与生产建议

- 将 CORS `allow_origins` 收紧为前端实际域名。  
- 对数据源 `url` 做白名单或鉴权，避免 **开放代理任意内网文件**（SSRF 风险）。  
- 大流量场景可结合 CDN、缓存响应头；TiTiler 官方亦有 [Performance Tuning](https://developmentseed.org/titiler/advanced/performance_tuning/)。

---

**权威路径与参数以当前部署的 `{BASE}/docs` 为准**；TiTiler 版本升级时，个别查询参数名称可能变化，以前端实际联调与 OpenAPI 为准。
